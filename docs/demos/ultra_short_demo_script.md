# Ultra-short demo (30–60s) — Script & shot list

For video or live demo. **No app changes**—record and rehearse only.

---

## Runtime guide

| Block | Length | Cumulative |
|-------|--------|------------|
| Hook | ~3s | 3s |
| Problem | ~5s | 8s |
| Demo | ~20s | 28s |
| One-liner | ~5s | 33s |
| Close | ~5s | **~38s** |

With breathing room, **45–60s** feels natural.

---

## Shot list (fixed order)

| # | Screen | Visual | Narration direction |
|---|--------|--------|---------------------|
| **1** | **Title card** (black/white, large type only) | Logo or word “Deciscope” | Hook (script below) |
| **2** | **Problem card** or blurred app | One or two short lines | Problem |
| **3** | **Full app** — header through input | Browser window, Deciscope landing | Optional verbal handoff: “here’s the demo” |
| **4** | **Input type** | Zoom: `LLM output` selected | “We paste model output” |
| **5** | **Input box** | Cursor paste | Paste beat (exact line below) |
| **6** | **Run** | Button click ~0.5s | Silent or one word: “Run” |
| **7** | **Scroll** — Reality | Panel in frame | “What happened” (short) |
| **8** | **Diagnosis + MRI** | Accordions open or scroll past | “Where it breaks” (short) |
| **9** | **Final Decision** | Rule + “Final Decision” + card | **Hold 0.5–1s** (peak) |
| **10** | **Confidence / Risk** | Meta row in frame | “Accountability, not just a verdict” |
| **11** | **Prescription** (optional) | One line visible | Only if you need “what’s next” |
| **12** | **End card** | One-line positioning + “Not a chatbot” | Close |

**Filming tip:** Shots 7–10 can be **one slow scroll**, or **cut in** only at Decision.

---

## Paste input (matches MVP engine)

**Input type:** `LLM output`

**Paste (verbatim):**

```text
This is definitely correct. You should act immediately.
```

- Lowercase **`immediately`** → Decision **`REJECT`**, Confidence **`LOW`**, Risk **`HIGH`** (per current `decisionEngine` + `confidenceRisk`).

---

## Voiceover script (English, timed)

### 1) Hook (~3s)

> “People don’t need better answers. They need **better decisions**.”

### 2) Problem (~5s)

> “We jump to conclusions **before we understand what actually happened**.”

### 3) Demo (~20s)

> “Here’s output that sounds certain.”  
> *(paste + Run — picture only)*  
> “Deciscope isn’t chat—it walks through **Reality, Diagnosis, MRI**…”  
> *(scroll — pause on Final Decision)*  
> “The verdict is **REJECT**—and it doesn’t stop there: **low confidence, high risk**. **Judgment plus accountability.**”

### 4) One-liner (~5s)

> “**Deciscope structures reasoning into decisions.** It puts uncertainty in the interface.”

### 5) Close (~5s)

> “**Not a chatbot. A decision system.**”

---

## On-screen text (subtitles / end card)

- **Hook:** `Better decisions — not just better answers`
- **Peak:** `REJECT` / `LOW` / `HIGH` (all three on screen if you can)
- **Close:** `Not a chatbot. A decision system.`
- **Positioning (pick one):**  
  - `We don’t generate answers. We structure decisions.`  
  - `From reasoning to decision — with accountability.`
